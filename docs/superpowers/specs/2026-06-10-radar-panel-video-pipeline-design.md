# 雷达面板视频生产系统设计

## 目标

构建一套可复用的视频生产系统，用于批量生成“战力评分 / 六维面板”类短视频。

系统目标不是只服务《三角洲行动》，而是把：

- 内容收集
- 评分数据整理
- 视频模板
- 动画选择
- 视频预览 / 输出

拆成固定流程，使后续切换到《瓦罗兰特》或其他题材时，只替换内容数据，不重做视频架构。

## 已确认范围

- 主分发形态：`9:16 竖屏`
- 当前示例题材：`三角洲行动`
- 当前示例系列：`烽火地带干员强度面板`
- 成片形式：`纯字幕 / 纯画面`
- 一条视频展示：`全部角色`
- 单角色时长：`5 秒`
- 结尾：`无总结，无收尾文案`
- 左侧：`人物图`
- 左上：`角色显示名`
- 右侧：`六维雷达图`
- 成片中：`不显示具体数值`
- 内部数据：`必须按数值维护`

## 核心设计原则

1. 内容层与视频架构彻底分离。
2. 同一套模板可复用于不同游戏或题材。
3. 动画效果从模板中独立出来，作为可选配置。
4. 输入方式采用填表驱动，本质上由结构化数据包驱动渲染。
5. Codex 负责前置信息整理与数据转换，不把评分逻辑散落在剪辑阶段。

## 系统分层

### 1. 内容层

内容层只描述“这次视频讲什么”，不描述视频怎么动。

每个角色至少包含：

- 标识 ID
- 正式名称
- 显示名称 / 外号
- 图片路径
- 六维分数
- 可选来源说明

内容层未来可以从《三角洲行动》切换为《瓦罗兰特》或其他题材，模板层无需修改。

### 2. 模板层

模板层定义固定视频架构：

- 左侧人物区域
- 左上名称区域
- 右侧雷达图区
- 每个角色页时长固定为 5 秒
- 所有角色顺序播放
- 整条视频播完直接结束

模板层不关心角色是谁，也不关心哪一维分数更高，只负责渲染结构。

### 3. 动画层

动画层从模板层中拆出，作为独立配置项。

推荐至少拆出这 3 类：

- 人物进场动画
- 雷达图数据展开动画
- 角色切换动画

动画不应写死在模板里，否则后续更换风格时必须重做整套视频。

### 4. 输出层

输出层负责把：

- 内容层数据
- 模板层结构
- 动画层配置

合成为最终预览或成片。

输出层不负责判断评分对不对，只负责正确呈现数据和动画。

## 当前页面结构

单角色页面固定为：

1. 左侧人物图
2. 左上显示名
3. 右侧六维雷达图

额外说明：

- 左侧人物优先使用透明底 PNG，仅保留人物主体
- 显示名优先使用玩家常用称呼；如果没有稳定共识，则回退到正式名
- 当前审核版名称策略中，仅 `疾风 -> 张姐` 明确使用外号，其余优先用正式名

## 六维定义

当前六维固定为：

- 信息
- 机动
- 压制
- 生存
- 功能
- 难度

### 判分标准

`信息`
衡量发现敌人、确认敌方位置、探测风险区域、建立视野优势的能力。

`机动`
衡量转点、拉枪线、抢身位、突进、脱战、快速位移的能力。

`压制`
衡量正面交火中的压迫能力，包括爆发、持续输出、逼退敌人、打乱敌方节奏的能力。

`生存`
衡量失误补救、保命、自救、规避致命伤、残局续航等能力。

`功能`
衡量对团队和局势的附加价值，例如控点、封路、辅助、破坏敌方节奏、创造战机。

`难度`
衡量角色操作与理解门槛。分数越高，表示越依赖时机、地图理解、连招意识和熟练度。

## 评分规则

- 内部统一按 `1.0 - 10.0` 打分
- 成片不展示数字，只展示雷达形状
- 评分必须以目标模式下的实战价值为准
- 当前示例视频中，评分语境限定为《三角洲行动》的 `烽火地带`
- 不按人气、剧情设定或其他模式表现打分

### 雷达图视觉映射建议

- `7 分以下`：明显收缩
- `8 分左右`：开始外扩
- `9 分以上`：可做显著尖角

这样即使不显示数字，观众仍能快速识别强项和短板。

## 单角色 5 秒时间轴

每个角色页固定 5 秒，建议时间轴如下：

### 0.0s - 0.3s

- 雷达图骨架已默认存在
- 左侧人物开始轻微进场
- 左上名称同步出现

### 0.3s - 1.2s

- 人物完成进场
- 雷达图数据面从中心向外展开
- 数据面动画与人物进场时间保持同步

### 1.2s - 4.4s

- 进入稳定展示阶段
- 雷达图保留轻微动态感，例如弱呼吸、轻微脉冲、弱流光

### 4.4s - 5.0s

- 快速切换到下一个角色
- 不做结尾总结或收口文案

## 动画配置拆分

动画层应支持选择式配置。

建议最低配置项：

- `characterEnter`
  - `slide-left-soft`
  - `fade-up-soft`
  - `zoom-in-soft`

- `radarBuild`
  - `scale-grow`
  - `radial-draw`
  - `pulse-build`

- `transition`
  - `hard-cut`
  - `flash-cut`
  - `blur-cut`

默认建议：

- 人物进场：`slide-left-soft`
- 雷达展开：`scale-grow`
- 切场：`hard-cut`

## 填表驱动的数据结构

建议一条视频对应一个 `video.json`。

```json
{
  "meta": {
    "title": "烽火地带干员强度面板",
    "series": "delta-force",
    "topic": "operators",
    "durationPerItem": 5
  },
  "theme": {
    "background": "dark-panel",
    "radarAxes": ["信息", "机动", "压制", "生存", "功能", "难度"]
  },
  "animation": {
    "characterEnter": "slide-left-soft",
    "radarBuild": "scale-grow",
    "transition": "hard-cut"
  },
  "items": [
    {
      "id": "jifeng",
      "name": "疾风",
      "displayName": "张姐",
      "image": "assets/operators/jifeng.png",
      "scores": {
        "信息": 6.5,
        "机动": 9.6,
        "压制": 7.8,
        "生存": 8.7,
        "功能": 6.2,
        "难度": 8.9
      },
      "sources": [
        "官方技能描述",
        "社区打法总结"
      ]
    }
  ]
}
```

### 结构规则

- `meta`：定义这条视频是什么
- `theme`：定义使用哪组雷达维度
- `animation`：定义本次套用哪组动画
- `items`：定义本次角色内容

### 数据约束

- `displayName` 允许为空；为空时回退到 `name`
- `scores` 固定为 6 维，不允许同模板内变更维数
- `image` 应优先使用透明底本地素材路径

## Codex 协助流程

Codex 负责前置内容整理，目标是把外部信息转成可直接灌入模板的数据。

固定流程如下：

1. 收集角色名单
2. 收集角色图片
3. 收集正式名称与玩家常用称呼
4. 收集技能与定位信息
5. 按目标题材语境整理六维分数
6. 输出结构化 JSON 数据

Codex 的职责不是直接剪辑，而是生成可复用、可审查、可回填的数据包。

## 使用流程

### 内容填充流程

1. 指定题材
2. 确定本条视频的角色列表
3. 由 Codex 收集角色信息和图片
4. 由 Codex 生成统一结构的数据包
5. 审核显示名、图片和六维分数

### 视频架构流程

1. 读取固定模板
2. 读取动画配置
3. 将 JSON 数据注入模板
4. 生成逐角色的 5 秒页面
5. 顺序拼接全部角色
6. 输出预览或成片

## 跨题材复用方式

当题材从《三角洲行动》切换到《瓦罗兰特》时：

- 保持模板层不变
- 保持动画层不变
- 只替换 `theme.radarAxes` 与 `items`

如果未来某个题材仍使用 6 维体系，则视频架构可直接复用。

## 当前不做的内容

- 不在当前设计中引入页面配置器
- 不在当前设计中引入手动逐帧剪辑流程
- 不在当前设计中支持不同视频使用不同维度数量
- 不在当前设计中加入结尾总结页

## 当前风险与约束

1. 不同题材的“玩家外号”稳定性差异很大，需要人工审核显示名。
2. 人物图片来源不统一时，透明底抠图质量会波动。
3. 如果后续题材需要 6 维以外的评分体系，当前模板需要扩展而不是直接复用。
4. 当前工作区不是 git 仓库，规格文档无法按流程提交版本控制。
